## Autonomous System (AS)
<mark>동일한 라우팅 정책으로 하나의 관리자에 의하여 운영되는 네트워크, 즉 한 회사나 단체에서 관리하는 라우터 집단</mark>을 자율 시스템(AS)라 하며, 
각각의 자율 시스템을 식별하기 위한 인터넷 상의 고유한 숫자를 망식별번호(AS번호)라 한다.

인터넷의 확산으로 네트워크의 크기가 커지고 라우팅 정보가 방대해지자, 전체 네트워크를 하나의 라우팅 프로토콜로 관리하는 것이 불가능해졌다. 
이에 따라 네트워크 관리범위를 계층적으로 체계화하고 단위 별로 라우팅 정보를 효율적으로 관리하기 위하여 AS가 도입되었다.

AS 개념이 도입된 이후에는 각 라우터가 인터넷 전체 경로를 모두 기억할 필요가 없어졌다.
기본적으로는 자신이 속한 AS 내부 경로 중심으로 정보를 유지하고, 다른 AS와의 연결 정보는 경계 라우터를 통해 전달받는다.
이 경계 역할을 수행하는 장비가 ASBR(Autonomous System Boundary Router)이며, 인접 AS의 경로를 라우터에 제공한다.

라우팅 대상이 내부냐 외부냐에 따라 프로토콜도 구분된다.
AS 내부 라우팅에는 IGP(Interior Gateway Protocol)를, AS 간 라우팅에는 EGP(Exterior Gateway Protocol)를 사용한다.
대표적인 IGP로는 RIP, IGRP, EIGRP, OSPF가 있고, EGP로는 BGP가 널리 쓰인다.

AS번호는 둘이상의 상위 ISP, IX와 연결된 다중 연결 사이트(Multi-Homed Site)를 구성하는 경우 필요하다. 
다중 연결 사이트에서는 도달가능정보를 여러 ISP에 배포하므로 라우팅 정책에 혼란을 준다. 
이런 경우 AS번호를 부여함으로써 AS가 자신의 라우팅 정책과 도달가능정보를 명확하게 관리할 수 있다.

AS번호를 사용하면 다음과 같은 운영상 장점을 가질 수 있다.
- 인터넷상에서 독립적인 네트워크를 식별
- 외부 네트워크와의 경로를 교환
- 고유한 라우팅 정책을 구현

https://asrank.caida.org/ 에서는 AS별 순위, 국가/기관 정보, AS 간 연결 관계(고객·피어·제공자) 같은 인터넷 토폴로지 정보를 확인할 수 있다.
KT의 AS번호는 4766, LG는 3786, SK는 9644이다.
