## Virtual Routing and Forwarding (VRF)

하나의 물리적인 라우터 내에서 여러 개의 독립적인 가상 라우팅 테이블을 동시에 운영하는 기술이다.  
👉 VRF를 쓰면 한 라우터에서 네트워크를 여러개로 분리할 수 있다.

```
[라우터]
 ├─ VRF-A (라우팅 테이블 A)
 ├─ VRF-B (라우팅 테이블 B)
 └─ VRF-C (라우팅 테이블 C)
```

## 특징

1.  완전한 분리 - 서로 다른 VRF는 기본적으로 통신 불가
2.  같은 IP 사용 가능 - overlapping IP 해결
3.  독립적인 라우팅 - 각 VRF마다 LPM 따로 적용됨

## 활용 사례

1. 통신사 (MPLS VPN)
   - 고객별 네트워크 분리
2. 기업 네트워크
   - 부서별 분리
   - 보안 구역 분리
3. 클라우드/멀티테넌시
   - 여러 고객 환경 분리

## VLAN vs VRF

| 구분                   | VLAN                     | VRF                  |
| ---------------------- | ------------------------ | -------------------- |
| 계층                   | L2                       | L3                   |
| 역할                   | 브로드캐스트 도메인 분리 | 라우팅 테이블 분리   |
| 목적                   | 네트워크 구역 나누기     | 라우팅 테이블 나누기 |
| IP 중복                | 불가능                   | 가능                 |
| 서로 다른 영역 간 통신 | Inter-VLAN Routing 필요  | Route Leaking 필요   |
